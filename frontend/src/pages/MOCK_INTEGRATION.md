# Mock Integration Documentation

This file explains how the frontend is wired to the mock hydraulic data server and how to switch back to real OPC UA/PLC data sources.

---

## How the Mock Works

- The frontend connects to a local WebSocket server (`ws://localhost:9000`) that streams fake hydraulic station data.
- All command functions (e.g., Bearing ON, Shaft ON) update local state and logs, simulating real operations.
- No real OPC UA/PLC is required; all data and responses are generated by the mock server and mock service.

---

## Switching Back to Real Data

1. **Remove/Comment Mock WebSocket Code**
   - Delete or comment out the WebSocket connection and state updates in `Assembly.jsx`.
   - Restore any code that reads from real OPC UA/PLC sources.

2. **Replace Mock Service with Real Service**
   - In `Assembly.jsx`, change imports from `MockOpcuaService` back to your real service (e.g., `AssemblyControlService`).
   - Update all function calls to use the real service methods.

3. **Restore Real Command Logic**
   - Remove or comment out any local state updates that simulate command responses.
   - Restore code that sends commands to the real backend/API.

4. **Test with Real PLC/OPC UA**
   - Ensure your backend is running and accessible.
   - Test all frontend functions to verify real data flow.

---

## Example: Switching Assembly.jsx

**From Mock:**
```js
import MockOpcuaService from '../services/MockOpcuaService';
// ...existing code...
const handleBearingToggle = async () => {
  setLastCommand("Bearing ON"); // mock
  addLog("Bearing ON → TRUE (mock)"); // mock
};
```

**To Real:**
```js
import AssemblyControlService from '../services/Assemblycontrol';
// ...existing code...
const handleBearingToggle = async () => {
  const response = await AssemblyControlService.runCommand('BEARING_ON');
  setLastCommand('Bearing ON');
  addLog(`Bearing ON → ${response.status}`);
};
```

---

## Final Notes
- The mock code is clearly marked with comments (`// mock`).
- To switch back, remove all lines with `// mock` and restore the original service and logic.
- This keeps your codebase clean and easy to maintain.
